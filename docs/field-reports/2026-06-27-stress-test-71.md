Can you tell me what the average heating degree days (base 65°F) have been at Burlington Airport (KBTV) for the winter months of December through February over the past 15 years? I just want to get a sense of how cold it typically gets up here so I can budget accordingly.

**Category:** degree days (heating, cooling, growing) query
**Difficulty:** medium
**Status:** error_recovered
**Answer Quality:** correct

### Issues

#### TypeError from incorrect argument usage in seasonal_summary()

- **Category:** wrong_api_usage
- **Severity:** medium
- **Description:** The agent hit a TypeError on its first call because it used 'variable' as a keyword argument when seasonal_summary() expected positional arguments. While the agent recovered on retry, this indicates the skill documentation doesn't clearly specify the function signature and argument types (positional vs keyword).
- **Suggestion:** Update SKILL.md or the function docstring for seasonal_summary() to explicitly show the function signature with parameter positions marked, e.g., 'seasonal_summary(station, start_year, end_year, variable, months)' and clearly indicate which args are positional-only vs keyword. Add a 'Common Mistakes' or 'Gotchas' section noting that arguments must be passed positionally, not as keyword arguments.
- **Pointer:** `SKILL.md - seasonal_summary function documentation`

#### Ambiguity in seasonal data interpretation

- **Category:** doc_gap
- **Severity:** low
- **Description:** The agent returned 'yearly_data' with years 2011-2025, but it's unclear from the output whether these represent calendar years or winter seasons (e.g., 'Winter 2015' could mean Dec 2014-Feb 2015 or Dec 2015-Feb 2016). The agent's answer text mentions 'Winter 2015' and 'Winter 2024' which suggests it's interpreting these as winter seasons, but the documentation doesn't clarify how seasonal data maps to calendar years.
- **Suggestion:** Add clarification in SKILL.md about how seasonal data is labeled and interpreted. For example: 'When requesting winter months (Dec-Feb), the returned year represents the year in which the winter season ends (e.g., Winter 2015 = Dec 2014 through Feb 2015).' Include an example in recipes.md showing how to interpret seasonal results.
