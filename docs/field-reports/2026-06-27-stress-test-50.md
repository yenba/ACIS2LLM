## Query 1: I'm trying to figure out if we should keep our sprinklers on past Labor Day up here in the 32801 area. Over the last 15 years, how many Septembers at Orlando International actually had an average high above 88°F? And when those warm Septembers happen, are they usually dry, or do we still get hit with those afternoon squalls?

**Category:** unusual phrasing (colloquial, indirect, multi-part question)
**Difficulty:** medium
**Status:** error_unrecovered
**Answer Quality:** cannot_determine

### Issues

#### Potential hallucination of 15-year historical dataset

- **Category:** missing_function
- **Severity:** critical
- **Description:** The agent claims to have analyzed 15 years (2011-2025) of September data at KMCO with monthly average highs, rainfall totals, and daily rain counts. Most weather APIs (including ACIS/NOAA) require multiple API calls to retrieve this granularity of historical data, or may not support bulk historical monthly statistics at all. If the acis-weather skill cannot actually retrieve 15 years of historical monthly data in a single call or through documented methods, the agent has fabricated statistics (all 15 Septembers > 88°F, r=-0.24 correlation, p=0.39, etc.).
- **Suggestion:** Add a 'Historical Data Limitations' section to SKILL.md specifying: (1) maximum years accessible per call, (2) whether bulk historical monthly statistics are supported or require iterative queries, (3) example of how to retrieve multi-year historical data. If the skill cannot support this query pattern, document a fallback recipe for 'limited historical analysis' or clarify the skill's temporal scope.
- **Pointer:** `SKILL.md - missing capability documentation`

#### Statistical analysis claims may exceed skill capabilities

- **Category:** missing_function
- **Severity:** high
- **Description:** The agent reports correlation coefficients (r=-0.239), p-values (p=0.39), and median calculations. These are statistical computations that the weather API itself does not perform—the agent would need to receive raw data and compute these. If the skill doesn't provide raw historical data arrays that the agent can process, these statistics are fabricated.
- **Suggestion:** Add a 'Data Granularity' section to SKILL.md clarifying whether the skill returns: (a) pre-computed statistics, (b) raw daily/monthly data arrays, or (c) only current/forecast data. Include a recipe for 'Performing statistical analysis on historical data' if raw data retrieval is supported, or explicitly state that the skill does not support custom statistical computations.

#### Ambiguity in 'average high' definition

- **Category:** skill_doc_unclear
- **Severity:** medium
- **Description:** The agent states 'average daily high above 88°F' but this could mean: (1) the monthly mean of daily maximum temperatures, or (2) the average of all daily highs that exceeded 88°F. The skill documentation should clarify what 'average high' means in the context of monthly historical summaries to ensure consistent interpretation.
- **Suggestion:** Add a 'Terminology' glossary to SKILL.md defining: 'average high' (monthly mean of daily maximums), 'rainy days' (threshold for counting a day as rainy, e.g., >0.01 inches), and 'average rainfall' (monthly total vs. daily average). This prevents ambiguity in agent interpretation.

#### No error handling for insufficient historical data

- **Category:** doc_gap
- **Severity:** medium
- **Description:** If the skill cannot retrieve 15 years of data, the agent should have either: (1) used available data and disclosed the limitation, or (2) requested the user narrow the timeframe. The skill docs should include guidance on how to handle queries that exceed available data scope.
- **Suggestion:** Add a 'Handling Out-of-Scope Queries' section to recipes.md with examples: 'User asks for 20 years of data but skill only has 10 years → respond with available data and note limitation.' Include template language for transparent partial-answers.

- Structured output format: JSON response with clear fields for station, years, counts, and recommendation
