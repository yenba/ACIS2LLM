Also, how many of those leap days actually saw measurable precipitation? I just want to know if we should pack for a mild spring day or if it's usually still pretty wintry.

### Issues

#### Incomplete historical data retrieval - only covered 17 of 25 requested leap years

- **Category:** wrong_api_usage
- **Severity:** high
- **Description:** The user explicitly asked for data from 'the last 25 leap years' (which would span approximately 1904-2024), but the agent only retrieved data from 1960-2024 (17 leap years). The agent listed 8 missing years (1928-1956) but never attempted to retrieve data from 1904-1924, which contains 7 additional leap years (1904, 1908, 1912, 1916, 1920, 1924). This is a significant data coverage failure.
- **Suggestion:** The skill documentation should include a recipe or example for handling queries that request data across multiple non-consecutive years (like all leap years). The agent needs to either: (1) make multiple API calls with date ranges covering each leap year, or (2) use a broader date range (1900-2024) and then filter results for Feb 29. The docs should clarify the correct approach for multi-year historical queries with specific date patterns.
- **Pointer:** `SKILL.md - missing recipe for multi-year historical queries`

#### No explanation for why earlier data wasn't retrieved

- **Category:** doc_gap
- **Severity:** medium
- **Description:** The agent reported missing years (1928-1956) but provided no explanation for why it didn't attempt to retrieve data from 1904-1924. The response doesn't clarify whether data from that earlier period is unavailable in the ACIS system or if the agent simply didn't try. This leaves the user without complete information.
- **Suggestion:** Add a gotcha or FAQ entry in the skill docs explaining the temporal coverage limits of ACIS data (e.g., 'ACIS data availability varies by station and time period; O'Hare data may not be available before a certain date'). Include guidance on how to detect and report when historical data is genuinely unavailable vs. when the query was just not broad enough.
- **Pointer:** `SKILL.md - missing gotcha about data availability time ranges`

#### Potential date filtering challenge not addressed

- **Category:** skill_doc_unclear
- **Severity:** medium

The agent may have used a date range that was too narrow (1960-2024) because it wasn't clear how to properly scope the query.
- **Suggestion:** Add a recipe example showing how to retrieve historical data for recurring dates (e.g., 'Get weather for all Feb 29 leap years'). This should demonstrate the proper date range parameter and how to filter results, or show how to make multiple calls for each leap year.
