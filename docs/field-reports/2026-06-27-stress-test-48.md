## Query 1: I'm planning a weekend trip to the 83702 area in Idaho, but I'm confused because it's so close to the state line and there are a few different weather stations nearby. Which station actually covers that ZIP code best, and what has the average high temperature been there for the last week of September over the past 10 years? I just need to know if I should pack a jacket or if it'll still feel like summer.

**Category:** ambiguous or tricky location
**Difficulty:** medium
**Status:** clean
**Answer Quality:** correct

### Issues

#### Backfill station ID (+) format not supported by xmacis2py.get_single_station_acis_data

- **Category:** doc_gap
- **Severity:** medium
- **Description:** The agent encountered a KeyError when using xmacis2py.get_single_station_acis_data with a backfill station ID containing a + character (KBOI+04115). While the agent recovered by switching to acis2llm.fetch_stations, the skill docs don't clearly document that backfill station IDs with + characters are incompatible with certain API functions. This forced an unnecessary retry.
- **Suggestion:** Add a Gotcha entry to SKILL.md: 'Backfill station IDs (containing + like KBOI+04115) are not supported by xmacis2py.get_single_station_acis_data. Use acis2llm.fetch_stations or acis2llm.find_best_station instead when dealing with backfill stations.' Also add a recipe example showing the correct workflow for backfill stations.
- **Pointer:** `SKILL.md - Gotchas section`

#### Station note references future date (2026)

- **Category:** data_quality
- **Severity:** low
- **Description:** The station_note field states 'data from 1875–2026' but 2026 is a future date. This suggests either the API is returning fabricated future data or there's a bug in how backfill station metadata is constructed. While this doesn't affect the core answer, it could mislead users about data availability.
- **Suggestion:** Investigate why the station metadata includes a future year. Either fix the metadata generation logic or add validation to prevent future dates from appearing in station notes.
- **Pointer:** `xmacis2py or acis2llm - station metadata generation`
