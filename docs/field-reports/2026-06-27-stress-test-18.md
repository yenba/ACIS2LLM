## Query 1: I'm trying to decide between visiting Aspen or Vail for a ski trip next month. Can you compare the average snowfall at KASE and KEGE for December over the past decade? I want to know which one usually gets more powder.

**Category:** multi-station comparison
**Difficulty:** medium
**Status:** clean
**Answer Quality:** correct

### Issues

#### Missing recipe for multi-station comparison queries

- **Category:** doc_gap
- **Severity:** medium
- **Description:** The skill lacks a dedicated recipe or example for comparing data across multiple stations over a time period. The agent had to figure out the workflow independently (query each station separately, then compare results), which worked but indicates a gap in documentation for this common query pattern.
- **Suggestion:** Add a multi-station comparison recipe to recipes.md showing the pattern: 1) Query metadata for all stations, 2) Retrieve data for each station, 3) Aggregate and compare results. Include example code for calculating averages, differences, and year-by-year comparisons.
- **Pointer:** `recipes.md`

#### Station substitution transparency could be improved

- **Category:** doc_gap
- **Severity:** low
- **Description:** The agent correctly substituted ground stations (050372, 058575) when airport stations (KASE, KEGE) had missing snowfall data, but the documentation doesn't provide guidance on when and how to perform this substitution. While the agent's note_on_original_stations field explains what happened, users may not understand why their requested stations were replaced.
- **Suggestion:** Add a gotcha or best practice note explaining that airport stations (K-prefix) often lack snowfall data in ACIS, and ground stations (5-digit IDs) are more reliable for snowfall queries. Include guidance on using find_best_station as a fallback strategy.
- **Pointer:** `SKILL.md or references/gotchas.md`

#### Unused station in stations_used list

- **Category:** data_quality
- **Severity:** low
- **Description:** The agent's output includes 'USS0006K39S' in stations_used but this station doesn't appear to have been used in the final calculation. This suggests either a failed attempt or incomplete cleanup, which adds noise to the output.
- **Suggestion:** Document that stations_used should only include stations that contributed to the final answer, or clarify when trial stations should be included for transparency purposes.

- The agent successfully handled the missing data error by using find_best_station to locate alternative stations with snowfall data
- The agent correctly calculated averages (27.95 and 24.5 inches) and year-by-year comparisons (7 vs 3 years)
