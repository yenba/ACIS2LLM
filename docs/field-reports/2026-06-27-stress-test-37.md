Can you check the historical records at O'Hare (KORD) and tell me which summer season (June through August) actually had the highest average high temperature on record? I want to know what year it was and what that average temperature was.

**Category:** cross-year ranking
**Difficulty:** medium
**Status:** error_recovered
**Answer Quality:** correct

### Issues

#### Wrong parameter name in seasonal_summary call

- **Category:** wrong_api_usage
- **Severity:** medium
- **Description:** Agent used 'variable' kwarg instead of 'parameter' in the first call to acis2llm.seasonal_summary, causing a TypeError. This suggests the skill docs are unclear about the correct parameter name for specifying what data to retrieve (e.g., high temperature).
- **Suggestion:** Add a clear parameter reference table in SKILL.md or recipes.md showing the exact kwarg names for seasonal_summary, especially distinguishing between 'parameter' (correct) and 'variable' (incorrect). Include a code example showing: seasonal_summary(station='KORD', parameter='tmax', season='summer')
- **Pointer:** `SKILL.md - parameter reference section`

#### Return type assumption error on single_station_meta

- **Category:** wrong_api_usage
- **Severity:** low
- **Description:** Agent assumed single_station_meta returns a dict but it returned a DataFrame, causing a KeyError. The agent recovered but this wasted a retry attempt.
- **Suggestion:** Add a Gotcha entry in SKILL.md: 'single_station_meta returns a pandas DataFrame, not a dict. Access columns with df['column_name'] syntax, not dict-style access.' Also consider adding a brief type annotation or return type description in the function documentation.
- **Pointer:** `SKILL.md - Gotchas section`

#### Missing recipe for cross-year historical ranking queries

- **Category:** missing_function
- **Severity:** medium
- **Description:** The query required iterating across many years (1959-2025), computing summer averages, and ranking them. There's no recipe showing how to handle this pattern of multi-year historical analysis with seasonal aggregation.
- **Suggestion:** Add a recipe in recipes.md titled 'Cross-Year Historical Ranking' showing: (1) how to get a list of years from station metadata, (2) how to loop through years calling seasonal_summary, (3) how to aggregate results into a ranked list. Include error handling for missing years.
