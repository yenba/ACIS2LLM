### Issues

#### Potential inclusion of incomplete/future year (2025) in historical average

- **Category:** data_quality
- **Severity:** medium
- **Description:** The agent included 2025 data (61.1°F) in the 10-year average calculation. If the query was run before October 24, 2025, this year's data for Oct 24-31 would not exist yet, making the average incorrect. The agent should validate that all years in the requested 'last 10 years' window have complete data for the specified date range before including them in calculations.
- **Suggestion:** Add a validation step in the skill documentation or recipe: before calculating multi-year averages, check the current date and exclude any year where the requested date range hasn't occurred yet. Alternatively, document that the API may return partial/future data and agents should filter it out. A gotcha entry could state: 'When requesting historical averages spanning multiple years, always verify that all years have complete data for the requested date range. Exclude future dates or incomplete current-year data from averages.'
