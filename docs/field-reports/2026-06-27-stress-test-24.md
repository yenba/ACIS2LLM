## Query 1: My friend claims that the summer of 2012 was the driest ever in Napa, but I'm not sure. Can you check the historical records at KAPC and tell me which summer season (June through August) actually had the least total precipitation on record, and how much rain they got that year?

**Category:** cross-year ranking
**Difficulty:** medium
**Status:** error_recovered
**Answer Quality:** partially_correct

### Issues

#### API parameter name confusion: 'variable' vs 'parameter'

- **Category:** wrong_api_usage
- **Severity:** medium
- **Description:** The agent used 'variable' instead of 'parameter' when calling seasonal_summary(), causing a TypeError. This was recovered from on retry, but indicates the skill docs don't clearly distinguish or document this parameter name. The parameter name 'parameter' is unintuitive and conflicts with the general concept of 'API parameter'.
- **Suggestion:** Add a gotcha entry in SKILL.md or the seasonal_summary reference doc explicitly stating: 'Note: The parameter is named "parameter" (not "variable"). Example: seasonal_summary(station="KAPC", parameter="precip", season="summer", year="2012")'. Consider adding this to a common mistakes section.
- **Pointer:** `SKILL.md or references/seasonal_summary.md`

#### Cross-year ranking pattern not documented

- **Category:** missing_function
- **Severity:** medium
- **Description:** The agent had to independently figure out how to perform a cross-year ranking query (calling seasonal_summary multiple times for different years and comparing results). This is a common query pattern for historical weather questions but has no recipe or guidance in the docs.
- **Suggestion:** Add a recipe to recipes.md titled 'Cross-year ranking or comparison' that shows how to: (1) identify the date range, (2) loop through years calling seasonal_summary or daily_summary, (3) collect and sort results. Include a code example for 'driest/wettest summer on record' queries.
- **Pointer:** `recipes.md`

#### Potential data quality issue with current/recent years

- **Category:** data_quality
- **Severity:** low
- **Description:** The answer includes 2024 and 2025 in the list of driest summers (0.00 inches). If the query was made before August 2025 was complete, these values may represent missing/incomplete data rather than actual zero precipitation. The agent didn't flag this uncertainty.
- **Suggestion:** Add a gotcha or note in the seasonal_summary docs: 'For current or recent years, verify the season is complete before interpreting results. Incomplete seasons may return partial or zero values.' Consider adding a check in the skill code to warn about incomplete seasons.

### What Worked

- The agent successfully recovered from the TypeError on retry, demonstrating the error handling works as intended
- The agent correctly identified and used find_best_station to resolve KAPC
- The agent correctly used seasonal_summary to retrieve summer precipitation data
- The agent provided a well-structured answer with specific years, rankings, and precipitation amounts
- The agent correctly answered the user's question (2012 was NOT the driest) and provided the actual ranking
