I'm just trying to gauge if we've been getting enough warm days lately to justify planting early, or if I should wait until June.

**Category:** degree days (heating, cooling, growing) query
**Difficulty:** medium
**Status:** error_unrecovered
**Answer Quality:** incorrect

### Issues

#### Missing data-fetching function calls - agent hallucinated GDD data

- **Category:** missing_function
- **Severity:** critical
- **Description:** The agent only reports using find_best_station and fetch_stations functions, but provides specific year-by-year GDD50 values (2016-2025). To calculate growing degree days, the agent needs to fetch daily temperature data (daily min/max or daily average temperatures) for April-May period across 10 years. The functions_used list is missing any daily data retrieval function, suggesting the agent hallucinated the GDD values rather than computing them from actual observations. This is a critical failure - the answer appears authoritative but is almost certainly fabricated.
- **Suggestion:** The skill needs either: (1) A dedicated function like acis2llm.fetch_daily_climate_data or acis2llm.get_growing_degree_days that can retrieve daily temperature data for date ranges, OR (2) Clear documentation showing how to use existing functions (like fetch_observations or similar) to retrieve daily data needed for GDD calculations. Without proper data-fetching capabilities, the agent cannot legitimately answer this query.
- **Pointer:** `SKILL.md - functions documentation`

#### No GDD calculation recipe or example in documentation

- **Category:** doc_gap
- **Severity:** high
- **Description:** Growing degree days queries are explicitly listed as a supported query category (based on the test metadata showing QUERY CATEGORY: degree days query). However, there's no recipe, example, or reference showing how to: (1) retrieve daily temperature data for multi-year periods, (2) calculate GDD from daily data using the formula: GDD = sum of max(0, (Tmax+Tmin)/2 - base_temp) for each day, or (3) aggregate GDD by month/season. The agent had no guidance on how to approach this calculation.
- **Suggestion:** Add a recipe to recipes.md titled 'Calculate Growing Degree Days' that shows: the complete function call sequence (station lookup → daily data fetch → GDD calculation), the GDD formula, and a worked example with sample data. Also add to SKILL.md under supported operations: 'Growing Degree Days: Use fetch_daily_observations to get daily min/max temps, then calculate GDD = Σmax(0, (Tmax+Tmin)/2 - base) for the period.'

#### Agent failed to recognize capability gap and hallucinated instead

- **Category:** agent_confusion
- **Severity:** high
- **Description:** The agent reported 'errors: none' and 'retries: 0' despite only calling station-finding functions. A properly functioning agent should have either: (1) called the appropriate data-fetching functions if they exist, or (2) failed gracefully and told the user 'I can find the station but don't have access to historical daily temperature data needed to calculate growing degree days.' Instead, it fabricated specific numbers (517.2 average, year-by-year breakdown) which gives false confidence in incorrect data.
- **Suggestion:** Add a gotcha to SKILL.md: 'GOTCHA: If you cannot find a function to fetch daily historical temperature data, do NOT hallucinate GDD values. Instead, inform the user that while you can identify weather stations, you lack access to the historical daily observations needed to calculate degree days.'

#### Station selection may be suboptimal for agricultural GDD queries

- **Category:** other
- **Severity:** low
- **Description:** The agent selected KMDW (Midway Airport). While this is a valid Chicago-area station, for agricultural GDD queries, rural or agricultural weather stations may provide more representative temperatures than urban airport stations (which experience urban heat island effects). The skill documentation should note this consideration.
- **Suggestion:** Add a note to SKILL.md: 'For agricultural queries (GDD, frost dates, growing season): prefer rural or agricultural stations over urban airport stations when available, as airport stations may have urban heat island bias affecting temperature readings.'
